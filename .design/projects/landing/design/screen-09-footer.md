## Accessibility

- **VoiceOver order:** column 1 label вҶ’ column 1 links in DOM order вҶ’ column 2 label вҶ’ column 2 links вҶ’ column 3 вҶ’ column 4 вҶ’ legal lines вҶ’ end.
- **Landmarks:** `<footer role="contentinfo">` is a landmark. Inside, no nested landmarks needed for this scale.
- **Heading structure:** **No headings inside the footer.** The column labels (`cГіdigo`, `colosseum`, etc.) are `<span>` or `<p>`, not `<h3>`. Reasoning: the footer is below the team `<h2>` and adding `<h3>` here would suggest a hierarchical relationship that does not exist (the footer is chrome, not a sub-section of "Team"). Use `<dl>` (description list) with `<dt>` for the label and `<dd>` for the link list, OR a `<nav aria-label="cГіdigo">` per group. Recommendation: `<nav aria-label="...">` per group вҖ” gives screen readers a clean "cГіdigo nav" landmark for each group.
- **External links:** all `target="_blank" rel="noopener noreferrer"`. Screen readers announce "opens in new tab" via the `target="_blank"` (or augment with `aria-label="..., abre em nova aba"` if desired).
- **Touch targets:** Mono 12px links wrapped with `padding: 4px 0` and `display: inline-block` for 8px effective vertical tap zone per link. With 8px gap between links (composed of 4px padding Г— 2), each link's effective tap zone is ~16px tall вҖ” below 24Г—24 AA. **Mitigation:** wrap each link in a `display: block` `<a>` with `padding: 8px 0`, increasing effective tap to ~28px tall. Width is column-bound, so target is wider than tall and passes 24Г—24 trivially.
- **Contrast:**
  - Column labels (`#555B66` on `#0E0F11`) = 3.16:1 вҖ” **fails AA at 11px**. Bump to `#8A8F99` = 6.46:1 AA.
  - Links (`#8A8F99` on `#0E0F11`) = 6.46:1 AA.
  - Legal lines (`#555B66` on `#0E0F11`) = 3.16:1 вҖ” **fails AA**. Bump to `#8A8F99` = 6.46:1 AA.
  - Hover amber (`#E8A020` on `#0E0F11`) = 6.92:1 AA.
- **Skip-links:** the page-level skip link points to `<main>`; from inside the footer, no further skip is needed because the footer is the last landmark.

## Image resources

**No imagery.**

The footer is text-only. Per the brand, social platform icons (X / Telegram / Discord) are tempting but forbidden вҖ” `imagery-style.md В§ Anti-Patterns` lists "blockchain iconography (hexagons, nodes, chains, circuit boards)" as forbidden, and consumer-app social icons read as "consumer financial app" which the brand also rejects. Use the platform name as text: `x.com/tgaprotocol`, `t.me/tgaprotocol`. The URL is the icon.

## Visual effects

- **Surface stacking:** footer stays on `var(--color-canvas)`. The 1px top border separates it from team. No surface step inside.
- **No texture.** The architectural grid lives only on the hero.
- **Hover amber on links:** 150ms ease-out `color` transition. The single sanctioned moment where amber appears in body text on hover.
- **No motion** beyond color transitions.

## Brand-contract checks

- вң… `border-radius: 0` вҖ” no boxed elements.
- вң… No shadows.
- вң… Amber pixel budget: zero amber at rest. Hover adds transient amber to links (single link at ~80Г—16 = ~1,280pxВІ, transient). Active locale + `tga.protocol` line at bottom вҖ” both `text-2` gray, not amber. Section contributes effectively 0pxВІ to the resting amber budget.
- вң… Three-layer hierarchy: Mono is dominant (column labels, links, legal). Geist appears only via the page-level `<h1>` already in the hero; the footer doesn't need its own Geist declaration. Inter is also absent вҖ” but the section is intentionally "chrome", not narrative content. The three-layer rule applies to **narrative sections**; the footer is allowed to be Mono-only as the chyron that closes the page. Per `STYLE.md В§ 2. Philosophy`: *"Three-layer hierarchy is the core law"* applies per screen, but the footer is a chrome element, comparable to the nav (which is also Geist + Mono only, no Inter). The page as a whole has the three layers; the footer is a Mono closing.
- вң… Tabular numerals: `2026` in copyright, `01-04` numerals in colosseum/copilot URL paths use `tabular-nums`.
- вң… Mono everything per the brief вҖ” Bloomberg Terminal density.
- вң… LGPD: the consent line lives in the capture forms, not the footer (per `content-strategy.md В§ Footer`). Footer carries the pre-launch disclaimer + copyright. No tracking-cookie banner вҖ” there is no client-side analytics planned for the launch (any future analytics вҶ’ server-side only, LGPD-compliant).
